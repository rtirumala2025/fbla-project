> **Purpose:** Guide for capturing high-quality screenshots for presentation materials  
> **Tools:** Browser DevTools, Screenshot extensions, or built-in OS tools

## Pre-Screenshot Setup

### Browser Preparation

1. **Clean Browser State**
   - Close unnecessary tabs
   - Clear browser cache (optional, for fresh look)
   - Use incognito/private mode for clean state
   - Set browser zoom to 100%

2. **Window Size**
   - Use standard resolution: 1920x1080 or 1280x720
   - Full-screen browser window
   - Hide browser bookmarks bar (optional)

3. **Demo Account**
   - Log in with demo account: `fbla-demo@example.com`
   - Ensure pet has appropriate stats (not all maxed)
   - Have sufficient coins (500+)
   - Pre-populate some transaction data

## Detailed Screenshot Instructions

### 1. Landing Page

**Steps:**
1. Navigate to `/` (homepage)
2. Wait for page to fully load
3. Scroll to show hero section and features
4. Capture full viewport

**What to Show:**
- Hero section with title
- Feature cards
- Navigation menu
- Call-to-action buttons

**Tips:**
- Use light theme for better presentation
- Ensure text is readable
- Hide any development tools

---

### 2. Pet Dashboard

**Steps:**
1. Log in with demo account
2. Navigate to `/dashboard`
3. Wait for pet stats to load
4. Ensure stats are visible (not all at 100%)

**What to Show:**
- Pet avatar/emoji
- All 5 stat bars (Health, Hunger, Happiness, Cleanliness, Energy)
- Action buttons (Feed, Play, Bathe, Rest)
- Coin balance
- Pet name and level

**Tips:**
- Use stats in yellow/orange range for visual interest
- Show at least one stat in red to demonstrate alerts
- Ensure coin balance is visible

**Variations to Capture:**
- Dashboard with low stats (showing alerts)
- Dashboard with high stats (happy pet)
- Dashboard with AI chat sidebar open

---

### 3. AI Chat Interface

**Steps:**
1. Open dashboard
2. Open AI chat sidebar/panel
3. Have a conversation with AI
4. Capture with 3-4 message exchanges visible

**What to Show:**
- Chat message history
- User messages (right-aligned, typically)
- AI responses (left-aligned, typically)
- Input field
- Send button
- Pet state information (if sidebar visible)

**Sample Conversation to Create:**
```
User: "How is my pet doing today?"
AI: "Your pet is feeling content! Current stats show..."
User: "/feed"
AI: "Great! I've fed your pet. Hunger increased..."
User: "Tell me about budgeting"
AI: "Budgeting is a key financial skill! Here's how..."
```

**Tips:**
- Show diverse message types (questions, commands, educational)
- Ensure AI responses are visible and readable
- Highlight command usage (`/feed`, `/status`)

---

### 4. Budget Advisor Analysis

**Steps:**
1. Navigate to analytics or budget advisor page
2. Submit sample transaction data (or use pre-loaded data)
3. Wait for analysis to complete
4. Capture results page

**What to Show:**
- Analysis summary (total spending, net balance)
- Top categories list
- Spending trends (with trend indicators)
- Overspending alerts (if applicable)
- Recommendations/suggestions

**Sample Data to Use:**
- 10-15 transactions across multiple categories
- Mix of amounts (some high, some low)
- At least one category showing overspending
- Clear trend indicators

**Tips:**
- Use data that shows interesting patterns
- Highlight overspending alerts if possible
- Show actionable recommendations
- Ensure all text is readable

**Variations:**
- Analysis with overspending (high severity alerts)
- Analysis with good spending patterns
- Trend analysis with increasing/decreasing trends

---

### 5. Shop System

**Steps:**
1. Navigate to `/shop`
2. Browse items
3. Add items to cart
4. Capture with cart visible

**What to Show:**
- Shop items grid/list
- Item details (name, price, description)
- Shopping cart with items
- Total price
- Purchase button
- Category filters (if applicable)

**Tips:**
- Show 2-3 items in cart
- Display total price clearly
- Use items from different categories
- Show sufficient coin balance

**Variations:**
- Shop with empty cart
- Shop with full cart
- Purchase confirmation screen

---

### 6. Analytics Dashboard

**Steps:**
1. Navigate to `/analytics`
2. Wait for data to load
3. Scroll to show all sections

**What to Show:**
- Coin balance summary
- Transaction history/list
- Charts or graphs (if implemented)
- Weekly/monthly summaries

- Leaderboard (if implemented)

**Tips:**
- Ensure data is visible and meaningful
- Show multiple transactions

### 7. Mini-Games

**Steps:**
1. Navigate to games page or specific game
2. Start a game

## Screenshot Tools & Techniques

### Browser DevTools

**Chrome/Edge:**
1. Press `F12` to open DevTools
2. Press `Ctrl+Shift+P` (Windows) or `Cmd+Shift+P` (Mac)
3. Type "screenshot"
4. Select "Capture full size screenshot"

**Firefox:**
1. Press `F12` to open DevTools
2. Click three-dot menu
3. Select "Screenshot" â†’ "Capture full page"

### Browser Extensions

**Recommended:**
- **Full Page Screen Capture** (Chrome)
- **Awesome Screenshot** (Chrome/Firefox)
- **Nimbus Screenshot** (Chrome/Firefox)

### OS Built-in Tools

**Windows:**
- `Win+Shift+S` - Snipping Tool
- `Win+Print Screen` - Full screen

**Mac:**
- `Cmd+Shift+3` - Full screen
- `Cmd+Shift+4` - Selection tool
- `Cmd+Shift+5` - Screenshot options

**Linux:**
- `Print Screen` - Full screen
- `Shift+Print Screen` - Selection
- Use `gnome-screenshot` or `scrot`

## Post-Processing Tips

### Image Editing

1. **Crop & Resize**
   - Remove unnecessary browser chrome
   - Focus on relevant content
   - Standardize dimensions

2. **Annotations** (Optional)
   - Add arrows to highlight features
   - Add text labels
   - Highlight important elements

3. **Format**
   - Use PNG for screenshots (better quality)
   - Compress if file size is an issue
   - Maintain aspect ratio

### Tools for Editing

- **Free:**
  - GIMP
  - Paint.NET (Windows)
  - Preview (Mac)
  - Canva (online)

- **Paid:**
  - Adobe Photoshop
  - Sketch
  - Figma

## Troubleshooting

### Issue: Screenshot too large
**Solution:** Use compression tools or resize to standard dimensions

### Issue: Text not readable
**Solution:** Increase browser zoom to 125% before capturing

### Issue: UI elements cut off
**Solution:** Use full-page screenshot or adjust window size

### Issue: Inconsistent appearance
**Solution:** Use same browser, theme, and zoom level for all screenshots
